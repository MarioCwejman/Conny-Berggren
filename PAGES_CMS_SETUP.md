# Pages CMS Setup

Det här repot är förberett för att använda Pages CMS som riktig editor.

## Det som redan finns

- `.pages.yml` i repo-roten beskriver editorn.
- `Production_First_Draft/content/site-content.json` innehåller allt redigerbart innehåll.
- `Production_First_Draft/index.html` och `Production_First_Draft/script.js` läser innehållet därifrån.

## Så slår du på editorn

1. Pusha repot till GitHub.
2. Se till att klientens GitHub-konto har skrivåtkomst till repot.
3. Öppna `https://app.pagescms.org`.
4. Logga in med GitHub.
5. Ge Pages CMS åtkomst till repot `MarioCwejman/Conny-Berggren`.
6. Öppna posten `Startsida`.
7. Ändra innehållet och spara.

## Hur klienten ska använda det

1. Öppna Pages CMS.
2. Klicka på `Startsida`.
3. Ändra fälten.
4. Klicka på `Save`.
5. Vänta på att webbplatsen publiceras om.

## Viktigt att veta

- Den lokala sidan `Production_First_Draft/editor-sketch.html` är nu en start- och hjälpsida, inte själva editorn.
- Den riktiga editorn ligger i Pages CMS.
- GitHub Pages eller Vercel kan fortfarande användas för själva webbplatsen.
- Pages CMS kräver GitHub-inloggning för att skriva tillbaka ändringar till repot.
